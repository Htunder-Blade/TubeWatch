# 实施计划

以下是分阶段计划；Phase 0 至 Phase 2 已完成真实验证，Phase 3 已实现并等待真实播放列表验证，后续阶段仍只是计划。

- **Phase 0：项目骨架和频道视频读取。** 建立 Python 包，通过 Notebook 手动读取公开频道最近视频。
- **Phase 1：保存检测状态并判断新增视频。** 使用 SQLite 记录每个来源已发现的视频；首次全部视为新增，重复检查去重。（已实现）
- **Phase 2：新增视频调用 TubeScribe。** 通过固定版本的外部 Python 包显式处理 pending 视频，记录成功或失败，不自动重试。（已实现）
- **Phase 3：支持播放列表。** 将标准 `/playlist?list=...` URL 作为另一种视频来源。（已实现，待真实网络验证）
- **Phase 3a：频道播放列表选择。** 自动列出频道页面公开展示的播放列表，在 Tester 中从频道视频或一个播放列表中选择单一检查来源。（已实现，待真实网络验证）
- **Phase 3b：隔离的端到端 Tester。** 对选中来源的 3 个精确视频执行 TubeScribe、展示字幕 sample，并由独立清理 cell 精确删除本次状态和产物。（已实现并完成部分真实验证；访问受限视频过滤待补充）
- **Phase 4：处理失败重试。** 为 `failed` 设计有边界的重试策略；`no_subtitles` 默认保持终态，不自动重试。
- **Phase 5：定时运行与通知。** 在核心流程稳定后增加 Windows 定时执行和通知。
